Description on how we used the start code

API-Documentation

Vi brugte startkoden som udgangspunkt til vores login JWT token system med user entities og roller samt error handeling og CORS filter. Oven på dette tilføjede vi yderligere Eitities og nogle facader hvor vi med brug af threads kunne fetche data fra de 5 brugte endpoints. Vi tilføjede nogle yderligere rolle begrænsede Endpoints i for af vores starwars API og gjorde i sammenhæng med dette også brug af openapi. Ud over brugerbegrænsning gjorde vi også brug af JPA og ændrede vores User entities til at kunne hashse deres password med et genereret salt. Vi tilfælde yderligere i forbindelse med deployment også et endpoint og tilhørende facade metoder til en createUser funktion så man også kunne oprette en bruger.

I forbindelse med frontend gjorde vi brug af routing til at kunne navigere mellem de forskellige view, samt et loginsystem fra et af ugens tidligere opgaver. Med dette kom også brugen af de udleverede JWT tokens som vi kunne bruge til brugeridentifikation samt at udlevere og reagere på brugerdefinerede indhold i henhold til rolle og navn i token payload. Brugen af denne valide token var også vigtig for at vide om en bruger var logget ud eller havde adgang og hvilken adgang til havde indhold som de havde. Vi brugte blandt andet dette til at vide en specifik bruger besked til den individuelle brugers rolle da de logger ind, samt til at give begge brugerne adgang til at de vores starwars end point som blev fremvist i form af nogle tables i længder defineret af det fetchede indhold fra vores starwars API.
